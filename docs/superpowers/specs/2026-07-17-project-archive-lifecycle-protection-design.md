# 项目档案启停与删除保护设计

## 背景

项目档案是 PMS 的基础资料，会被项目进度以及未来的售后、合同、订单等业务单据引用，也会同步到金蝶 ERP 或其他外部系统。基础资料一旦进入业务链路或外部系统，物理删除会造成历史单据断链、系统间数据不一致和审计争议。

当前系统只在删除项目档案时检查 `pms_project.archive_id`，尚未检查成功同步记录；前端仍始终展示删除入口，批量删除还会逐条执行，存在部分成功、部分失败的风险。现有档案 `status` 是未启动、进行中等业务生命周期字段，需求部门已确认暂不使用，不应与主数据启用状态混用。

## 目标

- 只有从未被业务单据引用且从未成功同步外部系统的档案才能物理删除。
- 不满足删除条件的档案可以禁用，禁用后保留全部历史数据和已有引用。
- 禁用档案不可编辑、不可同步、不可被新业务单据引用，允许有权限人员重新启用。
- 删除、禁用、启用和删除失败均有明确权限、中文原因和操作日志。
- 默认列表及业务下拉只显示启用档案，禁用档案仍可筛选查询。
- 建立统一引用检查入口，后续新增模块无需重新设计删除保护机制。
- 暂时隐藏未定义业务语义的原档案状态字段，但不删除数据库历史数据。

## 非目标

- 本次不删除 `pms_project_archive.status` 列，也不清理其历史值。
- 本次不建设可由用户配置的通用软删除平台。
- 本次不改变项目进度、ERP 同步和其他模块现有业务状态定义。
- 本次不允许字段规则把禁用档案重新开放编辑。

## 状态模型

`pms_project_archive` 新增 `is_enabled` 整数字段：

| 值 | 含义 | 行为 |
| --- | --- | --- |
| `1` | 启用 | 可按权限编辑、同步和被新业务引用 |
| `0` | 禁用 | 只读，不可同步，不可被新业务引用 |

- 字段非空，默认值为 `1`，现有档案迁移后全部为启用。
- 增加适合列表和下拉查询的索引。
- `is_enabled` 属于系统控制字段，不进入业务字段规则的可编辑范围。
- 现有 `status` 保留在数据库和数据字典中，并标记为“保留字段，暂未启用”。
- `status` 从项目档案列表、筛选、列设置、新增表单、编辑抽屉、字段规则和枚举管理中隐藏，后端不再接受业务页面修改该字段。

## 行为矩阵

| 档案情况 | 编辑 | 同步 | 新增引用 | 启停 | 删除 |
| --- | --- | --- | --- | --- | --- |
| 启用、无引用、无成功同步 | 允许 | 允许 | 允许 | 可禁用 | 允许 |
| 启用、已引用或曾成功同步 | 允许 | 允许 | 按现有业务约束 | 可禁用 | 禁止 |
| 禁用 | 只读 | 禁止 | 禁止 | 可启用 | 仅无引用且无成功同步时允许 |
| 正在同步 | 暂时锁定相关动作 | 正在执行 | 不改变 | 禁止禁用 | 禁止删除 |

禁用档案必须先重新启用才能修改。已有项目进度继续显示档案快照和引用信息，不因档案禁用而失效。重新启用后恢复编辑、同步和业务引用能力，但仍需遵守一档案只能建立一个项目进度等现有约束。

## 统一删除保护服务

新增项目档案使用情况服务，负责一次性收集删除阻止项，业务接口和列表投影不得分别实现不同规则。服务返回结构化结果：

```json
{
  "can_delete": false,
  "blockers": [
    {
      "type": "business_reference",
      "source": "project_progress",
      "label": "项目进度",
      "count": 2
    },
    {
      "type": "external_sync",
      "source": "kingdee",
      "label": "金蝶 ERP",
      "count": 1
    }
  ]
}
```

第一版检查器包括：

- `PmsProject.archive_id` 的项目进度引用数量。
- `PmsProjectArchive.erp_synced == 1`。
- `ErpSyncLog` 中该档案是否存在任意 `status == "success"` 的历史记录。
- 当前 `erp_sync_status == "pending"`，用于阻止同步期间禁用或删除。

成功同步历史是永久删除保护条件。后续同步失败不能解除已经发生的外部系统引用；只有失败同步记录且从未成功的档案仍可删除。

未来售后、合同、订单或其他外部系统引用项目档案时，必须向统一服务注册引用检查器。该规则同步写入 `AGENTS.md`，避免新模块绕过保护。

## 删除与批量操作

- 单条删除必须在事务内重新加载档案并实时执行删除保护检查。
- 有阻止项时返回 HTTP `409`，错误码为 `ARCHIVE_DELETE_BLOCKED`，同时返回中文阻止原因和 `suggested_action: "disable"`。
- 删除判断不能只依赖列表中的 `can_delete`，列表字段只用于改善交互。
- 数据库外键继续作为并发场景的最终防线。
- 批量删除改为后端专用接口，先校验全部档案；任意档案不可删除时整批不执行，避免部分删除。
- 批量启用、禁用执行同一权限和数据范围校验，并返回明确结果。
- `erp_sync_status == "pending"` 时禁止禁用和删除，提示等待同步完成。

## 权限模型

新增一个按钮权限：

```text
project:archive:toggle
```

菜单显示名称为“启用/禁用”。

- `project:archive:toggle` 仅控制启用与禁用。
- `project:archive:delete` 继续仅控制物理删除。
- 系统管理员和业务管理员模板在本次迁移中一次性获得新权限。
- 操作员模板默认不获得新权限。
- 自定义角色不自动授权，由管理员在角色管理中配置。
- 后端仍执行数据库实时 RBAC、数据范围和产品类别范围校验，前端隐藏不是授权依据。
- 后续启动不得反复补回被人工撤销的新权限。

## API 设计

### 档案列表

`GET /api/projects/archives/list` 增加：

- 查询参数 `enabled: bool | null`，前端默认传 `true`。
- 响应字段 `is_enabled`。
- 响应字段 `can_delete`。
- 响应字段 `delete_blockers`。

列表查询对当前页档案批量聚合项目引用和成功同步记录，禁止逐行查询造成 N+1。

### 启用与禁用

新增幂等接口：

```text
PUT /api/projects/archives/{archive_id}/enabled
```

请求体为 `{ "enabled": true | false }`。重复提交相同状态返回成功且不制造重复业务变更。禁用时检查是否正在同步，启用时继续执行数据范围和产品类别范围校验。

### 写入保护

- 更新档案接口发现档案已禁用时返回 `409 ARCHIVE_DISABLED`。
- ERP 单个及批量同步发现档案已禁用时返回 `409 ARCHIVE_DISABLED`。
- 档案 options 接口只返回启用档案。
- 项目新增接口即使被直接调用，也必须拒绝引用禁用档案。
- 查询禁用档案及其已有业务单据继续允许，确保历史可追溯。

## 前端交互

### 列表与筛选

- 原“档案状态”筛选替换为“启用状态”，默认值为“启用”。
- 支持“启用、已禁用、全部”三种查询。
- 增加紧凑的“启用状态”列，显示启用或已禁用标签。
- 已禁用行使用克制的浅灰背景，项目编号附近显示“已禁用”。
- 原档案状态从默认列、列设置和自定义筛选中移除。

### 单条操作

- 启用档案显示“编辑、同步、禁用、删除”。
- 禁用档案显示“查看、启用、删除”。
- 不可删除时保留删除入口但置灰，悬浮显示具体原因，如“已被 2 个项目进度引用，只能禁用”。
- 后端因数据变化拒绝删除时，页面展示服务端最新阻止原因，不静默转换为禁用。

### 编辑抽屉

- 禁用档案仍可打开抽屉查看完整资料。
- 抽屉头部显示“已禁用”标签。
- 所有字段只读，不显示保存和保存并同步按钮。
- 启用档案维持现有仅点击字段值进入编辑、统一底部保存的交互。

### 批量操作

- 增加批量禁用和批量启用能力，并按当前筛选状态显示合适动作。
- 批量删除由后端全量预检，失败时列出不可删除档案和原因。
- 批量操作不允许产生部分删除；启停结果必须明确展示成功数和失败原因。

## 操作日志

以下动作接入统一操作日志：

- 启用项目档案。
- 禁用项目档案。
- 成功删除项目档案。
- 因业务引用、外部同步或同步中状态导致的删除失败。
- 批量启用、禁用和删除。

成功启停记录 `is_enabled` 前后值；删除失败日志记录结构化阻止项，便于后续审计。列表查询和删除预检不记录日志。

## 迁移策略

- SQLite 和 MSSQL 2017 均以非空默认值 `1` 增加 `is_enabled`。
- 全新数据库直接创建新字段；已有数据库通过幂等升级逻辑增加字段和索引。
- 现有档案统一迁移为启用，不根据旧 `status` 推断禁用状态。
- 原 `status`、`archive_status` 枚举及历史值不删除，只从业务管理界面隐藏。
- 字段目录继续收录 `status`，说明改为“保留字段，暂未启用”；新增 `is_enabled` 并标记为系统控制字段。
- 新权限节点只进行一次模板授权迁移，后续启动不回补。

## 测试策略

### 后端

- 无引用、无成功同步档案可以删除。
- 项目进度引用阻止删除并返回数量。
- 任意历史成功同步记录阻止删除，即使最新一次同步失败。
- 只有失败同步记录不阻止删除。
- 正在同步的档案不能禁用或删除。
- 禁用档案不可更新、同步或被新项目引用。
- 禁用档案原有项目进度仍可查询。
- 重新启用后恢复允许的业务操作。
- 启停权限、删除权限、数据范围和产品类别范围分别生效。
- 批量删除全有或全无，不产生部分提交。
- 启用、禁用、删除和删除失败均生成可读操作日志。
- SQLite 与 MSSQL 初始化升级契约均覆盖新增字段和索引。

### 前端

- 默认只显示启用档案，筛选可查看已禁用和全部。
- 原档案状态不再出现在列表、表单、列设置、筛选、字段规则和枚举管理。
- 删除按钮状态和悬浮原因与后端返回一致。
- 禁用抽屉只读，不能通过键盘或直接请求入口触发页面保存。
- 项目进度新增下拉不显示禁用档案。
- 批量删除被阻止时完整展示档案及原因。
- 运行项目档案、项目进度、RBAC、字段目录、枚举管理、操作日志、标准列表契约测试及 `npm run build`。
- 浏览器验证启用、禁用、重新启用、删除保护、抽屉只读和筛选流程。

## 实施顺序

1. 先增加删除保护、启停行为和权限契约测试。
2. 实现数据库字段、幂等迁移和统一引用检查服务。
3. 接入删除、批量删除、启停、项目新增和 ERP 同步后端保护。
4. 更新角色权限、字段目录、字段规则和枚举管理可见性。
5. 调整项目档案列表、筛选、操作按钮、批量操作和只读抽屉。
6. 更新 `AGENTS.md`、`change.md` 并执行完整回归与浏览器验收。

## 验收标准

- 任何被业务单据引用或曾成功同步外部系统的档案都无法物理删除。
- 被保护档案可以禁用，禁用后只读且不能产生新的业务引用或外部同步。
- 默认业务界面只展示启用档案，历史业务仍能正确读取禁用档案。
- 删除和启停权限可以独立配置并由后端实时执行。
- 所有失败原因、批量结果和操作日志均使用可理解的中文信息。
