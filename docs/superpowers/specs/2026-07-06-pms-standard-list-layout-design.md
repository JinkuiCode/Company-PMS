# PMS 标准列表布局模块化设计

## 背景

项目档案列表已经形成了较清晰的列表体验：紧凑工具栏、快捷筛选、自定义筛选、AG Grid、外置横向滚动条和分页。项目进度相关页面仍各自维护这些结构，后续新增售后管理等列表页面时会继续复制代码。

## 目标

把列表页面拆成可复用的标准层，后续新增列表页面时只配置业务按钮、筛选字段、表格列和数据，不重新写布局和自定义筛选逻辑。

## 方案

- `PmsDataList.vue`：标准列表页面壳，提供 header、toolbar、filters、grid、pagination 插槽，并内置统一 AG Grid 外置横向滚动条容器。
- `PmsListFilters.vue`：标准筛选区，业务页面通过默认插槽放快捷筛选，组件统一提供“添加筛选 / 清空筛选”和自定义条件行。
- `useListFilters.ts`：通用筛选组合函数，提供字段配置、操作符、值判断和多条件筛选能力。

## 本次迁移范围

- 项目档案列表使用标准列表壳和标准筛选组件。
- 项目进度入口列表使用标准列表壳和标准筛选组件。
- 项目进度任务明细页补上同一套筛选能力，并使用标准列表壳承载 header、工具栏、筛选、表格。
- 不改变后端 API、任务自动保存逻辑、列定义和业务流程。

## 后续新增列表页面约定

新增列表页面优先采用：

1. 业务页面维护 `rowData`、`columnDefs`、接口调用和业务动作。
2. 用 `PmsDataList` 组织标准布局。
3. 用 `PmsListFilters` 放快捷筛选和自定义筛选。
4. 用 `useListFilters` 配置字段类型、选项和特殊取值逻辑。

## 验收

- 项目档案和项目进度页面不再手写自定义筛选 UI。
- 自定义筛选字段和匹配逻辑集中在 `useListFilters.ts`。
- 项目进度任务明细可以按任务名称、负责人、状态、进度、日期筛选。
- 前端构建通过，既有 UI 契约测试通过。
