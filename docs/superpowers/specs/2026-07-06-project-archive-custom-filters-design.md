# 项目档案自定义筛选设计

## 背景

项目档案列表当前只有关键字、产品线、状态三个固定筛选项。用户后续需要按负责人、产品类型、日期、同步状态等字段筛选，如果每次新增字段都改模板和筛选逻辑，会让维护成本持续上升。

## 方案

本阶段采用列表上方的“添加筛选”条件构建器，保留现有三个快捷筛选项。用户点击“添加筛选”后选择字段、条件和值，多个条件之间按“并且”组合。

暂不采用每列表头漏斗图标。原因是项目档案表格支持列拖拽、固定列和横向滚动，表头漏斗会增加表头视觉重量，也容易与现有列操作冲突。

## 范围

- 支持字段配置集中维护，新增筛选字段主要修改配置。
- 支持文本、下拉选项、日期三类字段。
- 支持包含、等于、不等于、早于、晚于、介于等常用条件。
- 保留关键字、产品线、状态快捷筛选。
- 不改后端 API，不做保存筛选方案，不影响 AG Grid 列布局保存。

## UI 原则

- 筛选器保持紧凑，作为筛选栏的第二行条件，不使用弹窗。
- 按钮和控件沿用 PMS 现有冷灰、靛蓝、8px 圆角风格。
- 只有存在自定义条件时展示“清空筛选”。
- 条件行支持键盘访问，删除操作有明确文字。

## 验收

- 用户可以在项目档案上方添加任意字段筛选条件。
- 多个自定义筛选条件会同时生效。
- 已有三个快捷筛选仍然可用。
- 前端构建通过。
