# PMS 列表与导航紧凑化设计

## 目标

在不改变业务接口、权限语义和列表数据结构的前提下，统一项目档案与项目进度的表格体验，并提升左侧导航在全部展开时的可扫描性。

## 导航方案

- 展开宽度由 `220px` 调整为 `184px`，折叠宽度继续使用 `64px`。
- 采用用户确认的方案 C：一级菜单保持图标、文字和展开箭头；二级菜单区域增加竖向导引线。
- 二级菜单高度调整为 `36px`，通过缩进、导引线和间距表达所属关系。
- 当前二级菜单继续使用靛蓝浅底，并在导引线上增加靛蓝圆点；当前一级菜单只强化文字颜色。
- 导航滚动能力保持不变，收窄后不得出现菜单文字裁切或与展开箭头重叠。

## 项目档案列表

- 所有 AG Grid 表头文字居中，数据单元格保持现有对齐方式。
- 工具栏移除“保存布局”和独立重置按钮，改为复用 `PmsListColumnPicker.vue` 的“列设置”。
- 项目编号、项目名称、选择列和操作列为固定列，不允许隐藏。
- 产品线、状态、负责人、产品类型、计划日期、审计字段和 ERP 同步字段均可显示或隐藏。
- 可配置字段按照“业务信息、计划信息、审计信息、ERP 同步”分组。
- 用户选择、列顺序、宽度、固定状态自动保存到当前用户、当前浏览器的 `localStorage`。
- 兼容旧版 `pms_archive_grid_layout_v2` 布局数据；首次读取后转换为新版结构。
- “恢复默认”恢复默认显示字段、顺序、宽度和固定状态。

## 项目进度操作列

- 有删除权限、需要显示“更多操作”时，操作列固定为 `88px`。
- 没有删除权限、仅显示“详情”时，操作列固定为 `60px`。
- 操作按钮组在单元格内居中，不改变详情和删除的权限判断或事件处理。

## 共享组件

- `PmsListColumnPicker.vue` 增加可配置的无障碍名称和“恢复默认”事件。
- 项目进度不传新参数时保持当前文案与行为，避免回归。

## 影响范围

- 仅修改 Vue、TypeScript、CSS、前端契约测试和 `change.md`。
- 不修改后端 API、数据库、OA、ERP、RBAC 权限码或操作日志。
- 不调整项目档案与项目进度的业务字段含义。

## 验收标准

- 项目档案表头与项目进度表头均居中。
- 项目档案可以通过列设置即时隐藏和恢复可配置列，刷新页面后偏好仍保留。
- 导航在 `184px` 展开宽度下可以清晰区分一级和二级菜单。
- 项目进度操作列没有明显空白，详情和更多操作仍完整可点击。
- 1366×768 和 1600×900 下无文字遮挡、横向滚动失效或分页错位。
