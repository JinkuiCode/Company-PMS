## 2. 功能概述

用 1-3 句话描述：
- **做什么**：核心业务目标
- **为谁做**：用户角色（项目经理、销售、部门经理、管理员）
- **为什么做**：解决什么问题

> 示例：为项目经理提供项目档案的增删改查功能，作为项目进度的基础数据来源，支持按编号、名称、状态筛选。

## 3. 数据库设计

### 3.1 新增表结构

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PK, 自增 | 主键 |
| code | NVARCHAR(32) | 唯一, 非空 | 业务编号 |
| name | NVARCHAR(128) | 非空 | 名称 |
| status | INT | 默认1 | 1=未启动 2=进行中 3=已完结 |
| created_at | DATETIME | 默认当前时间 | 创建时间 |
| updated_at | DATETIME | 自动更新 | 更新时间 |

### 3.2 修改现有表

| 表名 | 操作 | 字段 | 说明 |
|------|------|------|------|
| pms_project | 新增字段 | archive_id | FK → pms_project_archive |

### 3.3 索引建议

- 唯一字段自动建唯一索引，无需额外声明
- 高频查询字段（status, dept_id）建议加索引

## 5. 前端页面

### 5.1 页面结构

- [ ] 列表页（AG Grid 表格 + 自定义分页）
- [ ] 新增/编辑弹窗（Element Plus Dialog）
- [ ] 详情页（可选）

### 5.2 表格列定义

| 列名 | 字段 | 宽度 | 可编辑 | 特殊渲染 |
|------|------|------|--------|----------|
| 编号 | code | 140px | 是 | - |
| 名称 | name | 220px | 是 | 链接跳转 |
| 状态 | status | 100px | 是 | 胶囊标签（颜色区分） |
| 创建时间 | created_at | 170px | 否 | 日期格式化 |
| 操作 | - | 120px | 否 | 编辑/删除按钮 |

### 5.3 交互逻辑

- **新增**：点击"新增"按钮，弹窗填写表单
- **编辑**：双击行 or 点击编辑按钮
- **删除**：单行删除 + 批量删除（勾选后）
- **搜索**：输入框实时搜索（防抖300ms）
- **分页**：使用 `<CustomPagination />` 组件

### 5.4 UI 风格要求

- AG Grid 主题：`ag-theme-alpine wechat-table`
- 状态标签：胶囊形状，柔和背景色 + 深色文字
- 分页组件：左侧总数 / 中间页码按钮 / 右侧每页条数

## 7. 业务规则

列出关键业务逻辑：

1. **唯一性校验**：编号不可重复
2. **引用检查**：删除档案前检查是否被项目引用，有引用则禁止删除
3. **状态流转**：未启动 → 进行中 → 已完结（可暂停）
4. **自动填充**：选择档案后自动带出编号和名称
