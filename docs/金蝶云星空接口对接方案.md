# 金蝶云星空 ERP 接口对接方案

> **文档版本**：v2.0  
> **更新日期**：2026-06-08  
> **适用场景**：PMS 项目档案同步至金蝶云星空辅助资料  
> **参考来源**：OA 流程归档推送金蝶（物料保存）Java Action 源码

---

## 一、对接概述

### 1.1 业务目标

将 PMS 系统中的**项目档案**数据同步至金蝶云星空 ERP，实现：
- 项目立项后自动在金蝶创建「销售项目」类别的辅助资料
- 项目信息变更时同步更新金蝶数据
- 保证 PMS 与 ERP 项目数据一致性

### 1.2 数据流向

```
┌─────────────┐      Session + Save       ┌──────────────────┐
│  PMS 系统   │ ─────────────────────────→ │  金蝶云星空 ERP   │
│             │                           │                  │
│ 项目档案表   │  FNumber = project_code   │ T_BAS_ASSISTANT  │
│             │  FDataValue = project_name │ DATAENTRY        │
└─────────────┘                           └──────────────────┘
```

```json
  "lcid": 2052

```

成功响应判断：`LoginResultType == 1`，返回 Session Cookie。

### 2.3 辅助资料保存接口

```
POST {K3_URL}/Kingdee.BOS.WebApi.ServicesStub.DynamicFormService.Save.common.kdsvc
```

**FormID**: `BOS_ASSISTANTDATA_DETAIL`

请求参数（data 为 JSON 字符串）：
```json
{
  "formid": "BOS_ASSISTANTDATA_DETAIL",
  "data": "{\"NeedUpDateFields\":[],\"NeedReturnFields\":[],\"IsDeleteEntry\":\"true\",\"IsVerifyBaseDataField\":\"false\",\"ValidateFlag\":\"true\",\"NumberSearch\":\"true\",\"Model\":{\"FId\":{\"FNumber\":\"xsxm\"},\"FNumber\":\"PMS-2026-001\",\"FDataValue\":\"企业内部项目管理系统\",\"FDescription\":\"\",\"FSeq\":0}}"
}
```

**关键字段映射**：

| 金蝶字段 | 说明 | PMS 对应 | 必填 |
|---------|------|---------|------|
| `FId.FNumber` | 辅助资料类别 | 固定 `"xsxm"`（销售项目类别编码） | ✅ |
| `FNumber` | 编码 | `project_code` | ✅ |
| `FDataValue` | 名称 | `project_name` | ✅ |
| `FDescription` | 备注 | 可选 | - |
| `FSeq` | 显示顺序 | 0 | - |

### 2.4 辅助资料查询接口

```
POST {K3_URL}/Kingdee.BOS.WebApi.ServicesStub.DynamicFormService.ExecuteBillQuery.common.kdsvc
```

用于查询金蝶中是否已存在该编码的辅助资料，判断是创建还是更新。

### 2.5 金蝶数据库表结构（辅助资料）

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `T_BAS_ASSISTANTDATA` | 辅助资料主表 | FID |
| `T_BAS_ASSISTANTDATA_L` | 辅助资料名称表 | FNAME（筛选 `FNAME='销售项目'` 定位类别） |
| `T_BAS_ASSISTANTDATAENTRY` | 辅助资料子表（具体条目） | FNUMBER = project_code |
| `T_BAS_ASSISTANTDATAENTRY_L` | 子表名称表 | FDATAVALUE = project_name |

### 3.2 数据库扩展

#### 3.2.1 扩展项目档案表 `pms_project_archive`

| 新增字段 | 类型 | 说明 |
|---------|------|------|
| `erp_synced` | BIT | 是否已同步到 ERP，默认 0 |
| `erp_sync_time` | DATETIME | 最后同步时间 |
| `erp_entry_id` | INT | 金蝶辅助资料分录内码 |
| `erp_sync_status` | NVARCHAR(32) | success / failed / pending |
| `erp_error_msg` | NVARCHAR(512) | 同步失败时的错误信息 |

#### 3.2.2 同步日志表 `erp_sync_log`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | INT | 主键 |
| `source_id` | INT | PMS 项目档案 ID |
| `action` | NVARCHAR(16) | create / update |
| `status` | NVARCHAR(16) | success / failed |
| `error_msg` | NVARCHAR(1024) | 错误信息 |
| `created_at` | DATETIME | 记录时间 |

### 3.3 同步流程

```
用户在 PMS 创建/编辑项目档案
        │
        ▼
PMS 保存到本地数据库（erp_synced = 0）
        │
        ▼
调用 KingdeeClient.login() 获取 Session
        │
        ▼
调用 KingdeeClient.query() 查询该 project_code 是否已存在
        │
   ┌────┴────┐
   │         │
 存在      不存在
   │         │
   ▼         ▼
更新       创建
   │         │
   └────┬────┘
        ▼
更新 pms_project_archive 同步状态
写入 erp_sync_log
```

### 3.4 Python 代码参考（基于 OA 源码模式）

```python
# app/services/erp.py
import httpx
import json
import logging
from app.core.config import settings

logger = logging.getLogger(__name__)

class KingdeeClient:
    """金蝶云星空 WebAPI 客户端（Session 认证模式）"""
    
    def __init__(self):
        self.base_url = settings.K3_URL
        self.client = httpx.Client(timeout=30, verify=False)  # 内网可能无证书
        self._logged_in = False
    
    def login(self) -> bool:
        """登录金蝶获取 Session"""
        url = f"{self.base_url}/Kingdee.BOS.WebApi.ServicesStub.AuthService.ValidateUser.common.kdsvc"
        resp = self.client.post(url, json={
            "acctID": settings.K3_ACCT_ID,
            "username": settings.K3_USERNAME,
            "password": settings.K3_PASSWORD,
            "lcid": 2052,
        })
        data = resp.json()
        if data.get("LoginResultType") == 1:
            self._logged_in = True
            logger.info("金蝶登录成功")
            return True
        logger.error(f"金蝶登录失败: {data}")
        return False
    
    def save(self, form_id: str, model: dict) -> dict:
        """保存单据"""
        url = f"{self.base_url}/Kingdee.BOS.WebApi.ServicesStub.DynamicFormService.Save.common.kdsvc"
        payload = {
            "formid": form_id,
            "data": json.dumps({
                "NeedUpDateFields": [],
                "NeedReturnFields": [],
                "IsDeleteEntry": "true",
                "IsVerifyBaseDataField": "false",
                "ValidateFlag": "true",
                "NumberSearch": "true",
                "Model": model,
            })
        }
        resp = self.client.post(url, json=payload)
        return resp.json()
    
    def query(self, form_id: str, field_keys: list, filter_str: str = "") -> list:
        """查询单据"""
        url = f"{self.base_url}/Kingdee.BOS.WebApi.ServicesStub.DynamicFormService.ExecuteBillQuery.common.kdsvc"
        payload = {
            "FormId": form_id,
            "FieldKeys": ",".join(field_keys),
            "FilterString": filter_str,
            "OrderString": "",
            "TopRowCount": 0,
            "StartRow": 0,
            "Limit": 100,
        }
        resp = self.client.post(url, json=payload)
        return resp.json()
    
    def close(self):
        self.client.close()


def sync_project_archive(db, archive) -> dict:
    """同步单条项目档案到金蝶辅助资料"""
    client = KingdeeClient()
    try:
        # 1. 登录
        if not client.login():
            return {"status": "failed", "error": "金蝶登录失败"}
        
        # 2. 查询是否已存在
        existing = client.query(
            "BOS_ASSISTANTDATA_DETAIL",
            ["FEntryID", "FNumber"],
            f"FNumber = '{archive.project_code}'"
        )
        
        # 3. 构建 Model
        model = {
            "FId": {"FNumber": "xsxm"},
            "FNumber": archive.project_code,
            "FDataValue": archive.project_name,
            "FDescription": f"PMS同步 - {archive.product_type or ''}",
            "FSeq": 0,
        }
        
        # 4. 保存
        result = client.save("BOS_ASSISTANTDATA_DETAIL", model)
        
        # 5. 解析结果
        response_status = result.get("Result", {}).get("ResponseStatus", {})
        if response_status.get("IsSuccess"):
            return {"status": "success", "id": response_status.get("SuccessEntitys", [{}])[0].get("Id")}
        else:
            errors = response_status.get("Errors", [])
            error_msg = errors[0].get("Message", "未知错误") if errors else "未知错误"
            return {"status": "failed", "error": error_msg}
    
    except Exception as e:
        logger.error(f"同步异常: {e}")
        return {"status": "failed", "error": str(e)}
    finally:
        client.close()
```
