# 金蝶云星空 ERP 接口对接方案

> **文档版本**：v1.0  
> **创建日期**：2026-06-08  
> **适用场景**：PMS 项目档案同步至金蝶云星空

---

## 一、对接概述

### 1.1 业务目标

将 PMS 系统中的**项目档案**数据同步至金蝶云星空 ERP，实现：
- 项目立项后自动在金蝶创建项目辅助资料
- 项目信息变更时同步更新金蝶数据
- 保证 PMS 与 ERP 项目数据一致性

### 1.2 数据流向

```
┌─────────────┐         WebAPI         ┌──────────────────┐
│  PMS 系统   │ ─────────────────────→ │  金蝶云星空 ERP   │
│             │    POST /Save          │                  │
│ 项目档案表   │    (创建/更新)         │  项目辅助资料     │
└─────────────┘                        └──────────────────┘
```

### 2.2 支持的认证方式

金蝶云星空 OpenAPI 支持 **5 种认证方式**：

| 认证方式 | 说明 | 适用场景 | 推荐度 |
|---------|------|---------|--------|
| **AccessToken 认证** | 通过 appId + secret 获取 token | 服务端对接 | ⭐⭐⭐⭐⭐ |
| **JWT 认证** | 基于 JSON Web Token | 安全性要求高 | ⭐⭐⭐⭐ |
| **摘要认证 (Digest)** | 基于摘要算法 | 中等安全性 | ⭐⭐⭐ |
| **签名认证** | 请求签名校验 | 防篡改 | ⭐⭐⭐⭐ |
| **基本认证 (Basic)** | 用户名/密码 | 测试环境 | ⭐⭐ |

**推荐方案**：使用 **AccessToken 认证**，流程简单、安全性好、适合服务端对接。

### 2.3 核心接口

#### 2.3.1 认证接口

```
POST /k3cloud/Kingdee.BOS.WebApi.ServicesStub.AuthService.ValidateUser.common.kdsvc
```

请求参数：
```json
{
  "acctID": "账套ID",
  "username": "用户名",
  "password": "密码",
  "lcid": 2052
}
```

#### 2.3.2 单据保存接口（创建/更新）

```
POST /k3cloud/Kingdee.BOS.WebApi.ServicesStub.DynamicFormService.Save.common.kdsvc
```

请求参数：
```json
{
  "formid": "表单ID",
  "data": {
    "Model": {
      "FNumber": "编码",
      "FName": "名称",
      "FStatus": "状态"
    }
  }
}
```

#### 2.3.3 单据查询接口

```
POST /k3cloud/Kingdee.BOS.WebApi.ServicesStub.DynamicFormService.ExecuteBillQuery.common.kdsvc
```

用于查询金蝶中已存在的数据，判断是创建还是更新。

## 三、PMS 对接方案设计

### 3.1 技术架构

```
┌─────────────────────────────────────────────────────────┐
│                      PMS 后端                            │
├─────────────────────────────────────────────────────────┤
│  app/services/erp.py        # ERP 对接服务层            │
│  ├─ KingdeeClient           # 金蝶 API 客户端封装       │
│  ├─ authenticate()          # 认证获取 session          │
│  ├─ save_project()          # 保存项目到金蝶            │
│  └─ sync_project_archive()  # 同步项目档案              │
├─────────────────────────────────────────────────────────┤
│  app/api/erp.py             # ERP 对接 API 路由         │
│  ├─ POST /api/erp/sync      # 手动触发同步              │
│  └─ GET  /api/erp/status    # 查询同步状态              │
└─────────────────────────────────────────────────────────┘
```

### 3.2 数据库扩展

#### 3.2.1 新增配置表：erp_config

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键 |
| erp_name | NVARCHAR(64) | ERP 名称（如"金蝶云星空"） |
| api_base_url | NVARCHAR(256) | API 基础地址 |
| acct_id | NVARCHAR(64) | 账套 ID |
| username | NVARCHAR(64) | 用户名 |
| password_hash | NVARCHAR(256) | 加密后的密码 |
| form_id | NVARCHAR(64) | 表单 ID（项目辅助资料） |
| sync_enabled | BIT | 是否启用同步 |
| created_at | DATETIME | 创建时间 |

#### 3.2.2 扩展项目档案表：pms_project_archive

| 新增字段 | 类型 | 说明 |
|---------|------|------|
| erp_synced | BIT | 是否已同步到 ERP |
| erp_sync_time | DATETIME | 最后同步时间 |
| erp_project_id | NVARCHAR(64) | 金蝶中的项目 ID |
| erp_sync_status | NVARCHAR(32) | 同步状态：success/failed/pending |

#### 3.2.3 新增同步日志表：erp_sync_log

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键 |
| module | NVARCHAR(32) | 模块（project_archive） |
| action | NVARCHAR(16) | 操作（create/update/delete） |
| source_id | INT | PMS 中的 ID |
| target_id | NVARCHAR(64) | 金蝶中的 ID |
| status | NVARCHAR(16) | success/failed |
| error_msg | NVARCHAR(1024) | 错误信息 |
| created_at | DATETIME | 记录时间 |

### 3.3 同步流程

```
┌──────────────────────────────────────────────────────────────┐
│  1. 用户在 PMS 创建/更新项目档案                              │
└────────────────────┬─────────────────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────────────────┐
│  2. PMS 后端保存到本地数据库                                  │
│     - 设置 erp_synced = false                                │
│     - 设置 erp_sync_status = 'pending'                       │
└────────────────────┬─────────────────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────────────────┐
│  3. 异步调用金蝶 WebAPI                                       │
│     - 认证获取 session                                       │
│     - 查询金蝶是否已存在该项目（通过 FNumber）                 │
│     - 存在则调用 Update，不存在则调用 Save                    │
└────────────────────┬─────────────────────────────────────────┘
                     │
                     ▼
┌──────────────────────────────────────────────────────────────┐
│  4. 更新同步状态                                              │
│     - 成功：erp_synced = true, erp_sync_status = 'success'   │
│     - 失败：erp_sync_status = 'failed', 记录 error_msg       │
└──────────────────────────────────────────────────────────────┘
```

### 3.4 接口设计

#### 3.4.1 ERP 配置管理

```
GET    /api/erp/config          # 获取当前配置
POST   /api/erp/config          # 创建配置
PUT    /api/erp/config/{id}     # 更新配置
DELETE /api/erp/config/{id}     # 删除配置
POST   /api/erp/config/test     # 测试连接
```

#### 3.4.2 同步操作

```
POST   /api/erp/sync/project-archive/{id}   # 同步单个项目档案
POST   /api/erp/sync/project-archive/batch  # 批量同步
GET    /api/erp/sync/status/{id}            # 查询同步状态
GET    /api/erp/sync/logs                   # 查看同步日志
```

### 3.5 前端页面

#### 3.5.1 ERP 配置页

- 路径：`/system/erp-config`
- 功能：配置金蝶连接参数、测试连接、启用/禁用同步

#### 3.5.2 项目档案页扩展

- 新增"同步状态"列（显示 success/failed/pending）
- 新增"同步到 ERP"操作按钮
- 新增"批量同步"按钮

## 四、安全与权限

### 4.1 数据安全

| 措施 | 说明 |
|------|------|
| 密码加密存储 | 使用 bcrypt 加密存储 ERP 密码 |
| HTTPS 传输 | 金蝶 API 调用使用 HTTPS |
| 操作日志 | 所有同步操作记录到 erp_sync_log |
| 错误重试 | 失败后支持手动重试，最多 3 次 |

### 4.2 权限控制

| 操作 | 权限要求 |
|------|---------|
| 查看 ERP 配置 | 超级管理员 |
| 修改 ERP 配置 | 超级管理员 |
| 触发同步 | 项目经理 + 管理员 |
| 查看同步日志 | 项目经理 + 管理员 |

## 五、异常处理

### 5.1 常见异常场景

| 场景 | 处理策略 |
|------|---------|
| 金蝶 API 认证失败 | 记录错误日志，提示检查配置 |
| 网络超时 | 标记为 pending，支持手动重试 |
| 金蝶中已存在相同编码 | 转为更新操作 |
| 金蝶返回业务错误 | 解析错误信息，提示用户修正 |
| PMS 项目被删除 | 不自动删除金蝶数据（防止误删） |

### 5.2 错误码映射

| 金蝶错误码 | 含义 | PMS 处理 |
|-----------|------|---------|
| 0 | 成功 | 更新同步状态 |
| -1 | 认证失败 | 提示检查配置 |
| 其他 | 业务错误 | 记录详细错误信息 |
