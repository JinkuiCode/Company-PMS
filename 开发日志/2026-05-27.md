# 2026-05-27 开发日志

## 功能模块：项目档案管理

### 需求背景
在 PMS 系统中新增"项目档案"模块，作为项目的基础数据管理，与"项目进度"模块关联。

### 完成内容

#### 1. 数据库设计
- **新增表**: `pms_project_archive`（项目档案表）
  - `id` (INT, PK, 自增)
  - `project_code` (NVARCHAR(50), 唯一)
  - `project_name` (NVARCHAR(200))
  - `status` (INT, 1-未启动/2-进行中/3-已完结/4-暂停)
  - `manager_id` (INT, FK → pms_user)
  - `product_type` (NVARCHAR(100))
  - `created_at`, `updated_at`
- **修改表**: `pms_project` 新增 `archive_id` (INT, FK → pms_project_archive)

#### 2. 后端开发
| 文件 | 变更 |
|------|------|
| `backend/app/models/project.py` | 新增 `PmsProjectArchive` 模型，`PmsProject` 添加 `archive_id` 字段 |
| `backend/app/schemas/project.py` | 新增 Archive 相关 Schema（Create/Update/Response/Option） |
| `backend/app/services/project.py` | 新增档案 CRUD 函数，删除时检查引用 |
| `backend/app/api/projects.py` | 新增 `/projects/archives/*` 接口 |
| `backend/app/models/init_db.py` | 菜单迁移：项目列表→项目进度，新增项目档案菜单 |

**API 接口清单**:
- `GET /projects/archives/list` - 档案列表（分页+搜索）
- `GET /projects/archives/options` - 档案下拉选项
- `POST /projects/archives` - 创建档案
- `PUT /projects/archives/{id}` - 更新档案
- `DELETE /projects/archives/{id}` - 删除档案（有引用时禁止删除）

#### 3. 前端开发
| 文件 | 变更 |
|------|------|
| `frontend/src/views/project/ProjectArchive.vue` | 新建档案页面（AG Grid + 自定义分页 + CRUD弹窗） |
| `frontend/src/views/project/ProjectList.vue` | 进度创建改用档案下拉选择，统一分页风格 |
| `frontend/src/layout/AppLayout.vue` | 恢复深色侧边栏主题 |
| `frontend/src/router/index.ts` | 路由调整：项目列表→项目进度，新增项目档案 |

**UI 特性**:
- AG Grid 表格，企微风格样式覆盖
- 自定义分页组件（左侧总数 / 中间页码按钮 / 右侧每页条数）
- 双击行编辑，勾选框多选，批量删除
- 状态标签彩色圆角显示

#### 4. 菜单结构
```
├── 项目进度（原"项目列表"）
└── 项目档案（新增）
```

### 技术要点
- AG Grid 分页禁用（`:pagination="false"`），使用自定义分页组件
- 档案删除前校验 `PmsProject` 是否有引用该档案
- 项目创建时选择档案后自动带出 `project_code` 和 `project_name`
