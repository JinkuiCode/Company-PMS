# PMS 变更记录

## 2026-07-20 - 一键启动器版本一致性修复

- 重构根目录 `start-pms.command` 为稳定启动入口，通过 `.runtime/active-source` 明确选择前后端代码源；启动时打印代码路径、分支、提交号和数据库路径，配置失效时明确失败，不再静默回退旧版本。
- 启动器会检查 `8000/5174` 监听进程的工作目录和受管 PID；仅复用来源一致的进程，旧版本、手工进程或过期 PID 会先被停止再从当前代码源启动。
- 进程归属校验同时核对代码目录、实际监听 PID 和启动提交号；同一工作树切换提交后也会自动重启，避免继续复用旧代码进程。
- PID 文件改为记录实际监听进程，新增无副作用的 `PMS_STATUS_ONLY=1` 状态模式，日志统一写入根目录 `.runtime/logs`。
- 增加原子启动锁，重复双击会串行复用同一组服务；健康检查失败或启动被中断时，只清理本次拉起的临时进程和标记，成功后在终端等待关闭前主动释放锁。
- 本地 SQLite 数据迁移至工作区无关的 `.runtime/data/pms-dev.db`；首次建立时使用 SQLite 在线备份写入进程级临时文件，成功后原子替换，避免备份中断留下半成品数据库，也避免删除临时 worktree 时丢失测试数据。
- 已使用项目档案最新代码完成真实双击启动验证，前后端均保持运行，重复启动会复用同一组受管进程；合并后可直接切换到 `master` 并继续执行同样的来源校验。
- 新增 `frontend/tests/startup-launcher-contract.test.mjs` 并更新启动样式契约；契约会实际执行隔离状态检查，验证有效源码、失效指针和非法覆盖路径；启动器契约、样式契约、标准列表契约、系统 UI 一致性契约和根目录/当前代码源前端构建均通过。
- 浏览器验证管理员登录、项目档案启用/禁用操作和“项目档案编辑”抽屉均来自最新版本，页面控制台无错误。

## 2026-07-12 - 角色权限一致性与 RBAC 架构整改

- 新增统一实时授权上下文：每次请求从数据库计算有效用户、有效角色、权限并集、最宽数据范围和产品线并集；JWT 仅保留用户身份，撤权后下一次请求立即生效。
- 建立 `require_permission()` / `require_any_permission()` 统一依赖，将项目进度、档案、任务、ERP、用户、角色、部门、字典、字段、操作日志和仪表盘接入按钮级权限。
- 项目、任务、档案与 ERP 写操作同时校验对象数据范围和产品线；越界对象统一返回 `404`。修复用户未绑定部门时“本部门”范围意外退化为全部数据的问题。
- 已关联项目档案时，产品线权限以档案值为准；项目进度更新接口不再联动修改档案产品线，避免绕过档案编辑权限。
