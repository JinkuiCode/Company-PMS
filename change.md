# PMS 变更记录

## 2026-07-13 - 数据字典与枚举管理整改

- 新增只读字段目录服务与 `/api/field-catalog`，自动汇总 SQLAlchemy 模型、Pydantic Schema 和 126 个项目总表注册字段，支持关键词、模块、类型、来源和仅枚举字段筛选。
- 数据字典页面改为标准全宽密集列表，固定字段名称与编码列，提供横向滚动、分页、长说明提示和只读字段元数据展示。
- 将“字段管理”更名为“枚举管理”，迁移到 `/system/enum`，保留 `/system/field` 跳转兼容和原菜单/按钮 ID，避免破坏现有角色授权关系。
- 枚举管理只展示开发注册的五类业务枚举；产品线、产品类型可配置，档案状态、项目状态、任务状态锁定存储值；系统固定枚举不进入管理页。
- 删除五类无运行时用途的旧字段分类；未知历史自建分类保留数据库记录但不在枚举页面展示，并在初始化时输出迁移提示。
- 枚举值新增引用数量与删除保护；禁用值不再用于新建和变更，历史数据继续通过完整标签映射展示。
- 新增统一前端 `useEnumOptions` 能力，项目档案、项目进度、任务、仪表盘和角色产品线改为统一读取枚举，显示名变更后同步刷新共享缓存。
- 项目档案、项目进度、任务和角色产品线写接口接入统一枚举校验；所有枚举维护操作继续写入系统操作日志。
- 项目总表字段元数据增加 `enum_code`，详情抽屉和动态筛选按字段声明自动加载枚举，不再针对单一字段写特殊判断。
- 收紧角色数据范围边界：创建和更新只允许 1-4，全新数据库增加检查约束，历史异常值在运行时按最小权限处理。
- 新增字段目录、枚举管理前后端契约测试，并修复项目总表动态列表测试数据库路径不一致导致的残留数据问题。
- 修正数据字典全高布局，1366×768 与 1600×900 下横向滚动条和分页均完整可见；关键词输入支持 Enter 立即筛选。
- 加固 `start-pms.command` 的后台进程启动与端口回收逻辑，前后端使用 `nohup` 脱离终端输入，并在普通停止失败时强制清理占用端口的旧进程。
- 增加旧数据库升级契约，在任何角色 ORM 查询前补齐 `sys_role.product_lines`，避免已有数据库因 `create_all` 不会新增列而无法启动。
- 字段目录增加 SQLAlchemy 映射表自动兜底，ERP 同步日志、进度变更日志、菜单权限、枚举定义、关联表与登录令牌等模型字段全部纳入只读目录。
- 固定流程枚举初始化会清理未注册且未引用的旧值；仍被历史数据引用的旧值自动停用并保留标签，非数字脏值不再导致枚举页面报错。
- 产品线存储值禁止包含逗号，前后端同时校验，避免与角色产品线范围的序列化格式冲突。
- 项目进度与任务列表对可维护枚举标签统一进行 HTML 转义，避免异常显示名破坏表格单元格内容。
- 路由权限守卫改为返回跳转结果，移除 Vue Router 已弃用的 `next` 回调写法，保持权限导航行为不变并清理运行时警告。

## 2026-07-12 - 角色权限一致性与 RBAC 架构整改

- 新增统一实时授权上下文：每次请求从数据库计算有效用户、有效角色、权限并集、最宽数据范围和产品线并集；JWT 仅保留用户身份，撤权后下一次请求立即生效。
- 建立 `require_permission()` / `require_any_permission()` 统一依赖，将项目进度、档案、任务、ERP、用户、角色、部门、字典、字段、操作日志和仪表盘接入按钮级权限。
- 项目、任务、档案与 ERP 写操作同时校验对象数据范围和产品线；越界对象统一返回 `404`。修复用户未绑定部门时“本部门”范围意外退化为全部数据的问题。
- 已关联项目档案时，产品线权限以档案值为准；项目进度更新接口不再联动修改档案产品线，避免绕过档案编辑权限。
- 扩展 `/api/auth/me` 返回 `role_codes`、`permissions`、`data_scope`、`product_lines`；前端 Pinia、路由守卫、按钮、表格双击编辑和抽屉保存共用同一权限码。
- 角色权限保存自动补齐页面查看权限和祖先菜单，拒绝重复、不存在或已禁用的权限节点；取消查看时前端同步取消对应动作。
- 禁用角色不再参与授权、已关联用户的角色禁止直接删除，并保证至少一名有效用户保留用户管理与角色编辑能力。
- 启动初始化仅在全新数据库首次为默认管理员授权，既有数据库不再自动补回 `admin` 被撤销的权限。
- 新增服务器本地应急脚本 `backend/scripts/recover_role_permissions.py`，可按角色恢复指定权限或全部权限，并写入操作日志。
- 原公开的 SSO 测试链接生成器改为需要 `system:user:edit` 权限，生成行为写入操作日志；正式 OA SSO 登录与回调流程保持不变。
- 新增前后端 RBAC 契约测试，覆盖同一 JWT 即时撤权/恢复、禁用用户/角色、多角色并集、数据越界、初始化不回补权限、前端路由与操作入口。

## 2026-07-12 - 项目进度节点编辑与表头对齐

- 修复项目进度列表“节点”双击编辑时的状态选项类型不一致问题：列表数据和保存接口继续使用状态数值 `1/2/3`，下拉选项通过列格式化展示“进行中 / 已完结 / 暂停”，从而正确匹配当前值。
- 将项目进度列表的普通字段表头、分组表头和动态字段表头统一居中；不改变数据单元格既有对齐方式。
- 新增前端契约测试，防止节点编辑重新出现数值/中文标签错配，以及表头居中规则遗漏。

## 2026-07-12 - 项目进度整格编辑

- 移除项目进度列表编辑状态下由 AG Grid 单元格、选择器和输入框叠加形成的多重边框。
- 编辑器保持铺满原单元格，仅由单元格提供一层浅色焦点边界，避免出现嵌套的小输入框。
