PMS 系统（前端 + 后端）需要配合这个新的流程，彻底抛弃原来依赖 iframe 存 Cookie 的幻想。
1. PMS 前端改造 (oa-login-page 路由)
当页面加载时，拦截并解析 URL 参数：
读取 URL 中的 sso_login_id（和安全签名）。
不要显示账号密码输入框，直接在后台静默发起一个特殊的 Axios 请求（例如 POST /api/auth/sso-login），把 sso_login_id 传给 PMS 后端。
拿到后端返回的 access_token 后，存入 LocalStorage，然后直接 router.push 跳转到 PMS 首页。
2. PMS 后端改造 (新增 SSO 登录接口)
新增一个专门给 OA 用的免密登录接口：
接收前端传来的 sso_login_id。
校验逻辑：验证时间戳是否过期（防重放攻击），验证签名是否正确（防伪造）。
校验通过后，直接在 PMS 数据库中查询该 loginId 对应的用户。
直接为该用户签发 JWT Token（或 Session），返回给前端。
最终业务流：
用户登录 OA -> 点击 PMS 菜单 -> iframe 加载 pms_sso.jsp -> JSP 瞬间拿到账号并 302 重定向到 PMS 前端 -> PMS 前端拿到账号静默请求 PMS 后端 -> 登录成功渲染 PMS 页面。
